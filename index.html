Project: EFRS

Economic Facial Recognition System

Overview & Description

A problem we face while doing camera installations at festivals and other large events is if there is an incident at a gig
all of the footage from a 20+ cameras has to be manually searched using an NVR program like Milestone or GeoVision.
As this is a very time consuming process it would be a great help to be able to put a person of interests photo into a plugin
Program and have it show every frame that face shows up in order of the time stamp each frame has. this would be not only be
faster and less hands on it would create a difinitive timeline of events which is very important when it comes to investigations.

Facial recognition is not new or cutting edge but I believe it is a tool that should be more accessible to startups and small companies
who wish to use it to make safer work environments and collect data to further improve their efficiency.

Security is not the only application (pardon the pun) as there is a program we currently use at work that reads a face and gives feedback as to
The persons mood, disposition, tiredness, etc. This is excellent for passive feedback so you no longer have to send out surveys and
and pester clients for feedback as the customer satisfaction rate can be tracked and analysed unintrusively. As a bonus it would
piggyback right into the security camera system killing two birds with one stone.
The downsie is the cost, currently it is around $100 per month to have the software alone + hardware
so for a startup or small business this technology is relatively price prohibitive.
My goal is to write a facial recognition program that could be utilised as a platform
for an all in one program that would ideally be user friendly, intuitive and flexible.

Skills Tools & Tech Required:

Programming, coding and app design, Knowledge of hardware and sofware compatibility / capability.
I believe there is an open source platfrom that does facial recognition so finding that as a base to build off
and then packaging it so it could be mobile or desktop compatible would be the overall goal.

Outcome:

The success of this project will be measured by a balance of cost, form factor, and functionality.
My hope is that it could be less than $200 total to purchase, able to run on machines as economical as a Raspberry Pi,
and be able to put together a timeline of video footage using an input photo & deliver statistical environment data
with parameters such as mood, happiness, energy, and satisfaction.
